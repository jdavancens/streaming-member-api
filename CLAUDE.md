## What This Is

A high-fidelity model of Netflix's Member API: a **Consumer Edge GraphQL** supergraph composed with **Apollo Federation v2**. Five Spring Boot + Netflix DGS subgraph services sit behind Apollo Router. The API paradigms implemented are **client-driven GraphQL** (standard field selection) and **Server-Driven UI (SDUI)** (discovery-service returns component trees, the client renders whatever the server says).

## Commands

```bash
# Local dev (requires rover CLI installed: curl -sSL https://rover.apollo.dev/nix/latest | sh)
make dev                   # compose supergraph then docker-compose up

# Compose supergraph manually from SDL files in schema/
make compose               # writes services/router/supergraph.graphql

# Validate schemas (SDL + TypeSpec)
make schema-check          # rover subgraph check per subgraph
make spec-check            # tsp compile --no-emit

# Build all services
mvn clean package -DskipTests

# Build a single service
mvn clean package -pl services/member-service -am -DskipTests

# Run tests for a single service (Testcontainers spins up real deps)
mvn test -pl services/member-service

# Terraform (dev environment)
cd infrastructure/terraform/environments/dev
terraform init
terraform plan -var-file=example.tfvars
terraform apply -var-file=example.tfvars
```

## Architecture

```
Devices → Apollo Router (port 4000)
           ├─ member-service      :8081  → Amazon Keyspaces (Cassandra)
           ├─ billing-service     :8082  → RDS MySQL
           ├─ profile-service     :8083  → Amazon Keyspaces (Cassandra)
           ├─ entitlement-service :8084  → ElastiCache Redis
           └─ discovery-service   :8085  (SDUI; no persistent storage)
                     ↕
              Amazon MSK (Kafka)
              Topics: member.events, subscription.events
```

## Schema-First Development

**GraphQL SDL files in `schema/` are the source of truth.** Each service's `pom.xml` copies them into `target/classes/graphql/` via Maven Resources Plugin — DGS reads from there at runtime. Never edit the SDL files inside a service; edit in `schema/` and rebuild.

TypeSpec in `specs/` covers REST auth endpoints (`auth.tsp`) and Kafka event contracts (`events/*.tsp`). Run `tsp compile` inside `specs/` after changing these.

## Service Patterns

Every subgraph service follows the same DGS pattern:
- `@DgsComponent` classes contain `@DgsQuery`, `@DgsMutation`, and `@DgsData` annotated methods
- `@DgsEntityFetcher(name = "Member")` handles Federation entity resolution — the Router calls this when it needs to stitch `Member` fields contributed by a different subgraph
- `application.yml` reads all config from env vars with local defaults

## Key Files

| File | Purpose |
|---|---|
| `supergraph.yaml` | Rover composition config — maps each subgraph SDL to its routing URL |
| `services/router/router.yaml` | Apollo Router config (CORS, header propagation, health check) |
| `services/router/supergraph.graphql` | Generated by `make compose` — gitignored, not committed |
| `SCHEMA_CHANGELOG.md` | Required entry for every schema PR (breaking/additive/deprecation) |
| `specs/auth.tsp` | TypeSpec spec for REST auth endpoints (login, refresh, logout) |
| `specs/events/*.tsp` | Kafka event contracts for member.events and subscription.events |

## Federation Entity Resolution

`Member` is owned by `member-service`. Other subgraphs contribute fields to it:
- `billing-service` adds `Member.subscription`
- `profile-service` adds `Member.profiles`
- `entitlement-service` adds `Member.canStream`

Each of these services must implement `@DgsEntityFetcher(name = "Member")` and return the fields their subgraph is responsible for. The Router assembles the full `Member` object across subgraphs at query time.

## Entitlement Stream Slots

`entitlement-service` uses Redis Sets with TTL to enforce concurrent stream limits. Key: `entitlement:{memberId}:streams`. Each slot also gets a per-slot TTL key (`entitlement:{memberId}:slot:{streamId}`). The client must call `heartbeatStream` every 60 seconds or the slot expires after 90 seconds. Plan limits (maxStreams) are cached in-memory and updated by consuming `subscription.events` from Kafka.

## Infrastructure

Terraform modules are in `infrastructure/terraform/modules/`. The dev environment wires them together in `infrastructure/terraform/environments/dev/main.tf`. GitHub Actions uses AWS OIDC (no static credentials) — the IAM OIDC role ARN is output by `terraform apply` and must be set as `AWS_OIDC_ROLE_ARN` in GitHub secrets.
